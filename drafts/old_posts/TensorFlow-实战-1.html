TensorFlow入门

本节主要会介绍Tensorflow里最基本的三个概念——计算图(tf.Graph), 张量(tf.Session)和会话(tf.Session). 最后介绍了如何使用 tensorflow 来实现神经网络的训练过程并给出了一个代码样例.

TensorFlow计算模型-计算图

计算图上的每一个节点都是一个运算, 而计算图上的边则表示了运算之间的数据传递关系. 计算图上还保存了每个运算的设备信息(比如是通过CPU上还是GPU运行)以及运算之间的依赖关系. 计算图提供了管理不同集合的功能, 并且 tensorflow 会自动维护五个不同的默认集合.Tensor表达了数据结构，Flow表达了计算模型.

使用计算图有两个阶段，一是定义计算图中的所有计算，二是执行计算。在tensorflow中，系统会自动维护一个默认的计算图，通过tf.get_default_graph函数可以获取当前默认的计算图。TensorFlow支持通过tf.Graph函数来生成新的计算图，不同计算图上的张量和运算都不会共享。

通过Tensorflow训练神经网络模型

我们通过反向传播算法训练 NN，其实现了一个迭代的过程。在每次迭代的开始，首先选取一小部分训练数据，叫做一个batch,然后这个batch的样例会通过前向传播算法的到神经网络模型的预测结果. 因为训练数据都是有正确答案标记的, 所以可以计算出 当前神经网络模型的预测答案与正确答案之间的差距. 最后基于这预测值和真实值之间的差距, 反向传播算法会相应地更新神经网络的取值,使得在这个batch上 NN 模型的预测结果和真实答案更加接近.

一般来说,一个神经网络的训练过程会需要经过几百万甚至几亿轮迭代,这样计算图就会很大,而且利用率低. 如果我们的输入数据 X 用tf.constant输入的话不太好.所以TF提供了placeholder机制, 用于提供输入数据. placeholder相当于定义了一个位置,这个位置中的数据在程序运行时再指定. 这样在程序中就不需要生成大量常量来提供输入数据, 而只需将数据通过palceholder传入计算图.

深层神经网络

深度学习与深层神经网络

线性模型具有局限性, 只通过线性变换，任意层的全连接神经网络和单层神经网络模型的表达能力没有任何区别. 深度学习强调非线性, 这样模型的表达能力更强, 因此我们引入激活函数, 用以实现非线性表达. 同时加入隐藏层, 得以实现更强大的表达能力. 因此, 激活函数的引入和多层是现代深度学习模型的特点.

tensorflow 提供的激活函数主要有：

  • tf.nn.relu
  • tf.sigmoid
  • tf.tanh

损失函数的定义

经典损失函数有交叉熵和 Mean Square Error，前者用于分类问题，后者用于回归问题。

  • softmax
    我们常常通过Softmax层将神经网络输出变成一个概率分布. 然后就能通过交叉熵计算预测的概率分布和真实概率分布之间的差异了. Softmax函数，或称归一化指数函数，是逻辑函数的一种推广。它能将一个含任意实数的 K 维的向量$$z$$的“压缩”到另一个K维实向量$$\sigma(z)$$ 中，使得每一个元素的范围都在 (0,1) 之间，并且所有元素的和为1。该函数的形式通常按下面的式子给出：

$$Softmax(z)_j = \frac{e^{z_j}}{\sum_{k=1}^K e^{z_k}} \quad for\ j=1,\dots,K.$$

  • 交叉熵

    如何判断一个向量与期望的向量有多接近？使用 Cross Entropy。交叉熵刻画了两个概率分布之间的距离。交叉熵是信息论中的概念，原本用来估计平均编码长度。通过交叉熵计算预测的概率分布和真实答案的概率分布之间的距离。交叉熵是不对称的.

  • Mean Squared error
    回归问题得到的是具体的预测值, 所有对于回归问题, 最常用的损失函数就是均方误差. 定义如下:

$$ MSE(\hat y,y ) = \frac{\sum_{i=1}^n(\hat y_i-y_i)^2}{n}$$

  • 自定义损失函数
    实际问题的需要，我们可能需要自定义损失函数，例如我们可以定义下面的损失函数：

$$$
f(x,y) = \left {
\begin{array}{lr}
a(x-y) \quad x>y \
b(y-x) \quad x \leq y \
\end{array}
\right.
$$$

$$ Loss(y, \hat y) = \sum_{i=1}^n f(y_i, \hat y_i) $$

学习率的设置

learning rate 设置过大会导致在最有点左右振荡, 设置过小会导致学习速率过慢. 为了解决学习率的设置问题, tensorflow提供了一种更加灵活的学习率设置方法——指数衰减法: tf.train.exponential_decay。通过这个函数，可以先使用较大的学习率来快速得到一个比较优的解，然后随着迭代的继续逐步减少学习率.

过拟合问题

过拟合指的是当一个模型过于复杂之后，它可以很好地记忆每一个训练数据中随机噪音的部分而忘记了要去“学习”训练数据中通用的趋势。解决过拟合问题可以使用正则化，在损失函数中加入刻画模型复杂程度的指标，即优化$$J(\theta) + \lambda R(w)$$, 其中$$R(w)$$刻画模型的复杂程度，而$\lambda$表示模型复杂损失在总损失中的比例。

  • L1 正则化：$$R(w) = {\lVert w \rVert}_1 = \sum_i |w_i|$$.

  • L2 正则化：$$ R(w) = {\lVert w \rVert}_2^2 = \sum_i |w_i|^2$$.

  • 实践中，可以将L1正则化和L2正则化结合起来：$$R(w) = \sum_i \alpha|w_i|+(1-\alpha)w_i^2$$.

L1正则化 L2正则化
会让参数变得稀疏（会有更多的参数变为0）,从而达到特征选取的功能。 不会让参数变得稀疏，当参数很小时，比如0.001，这个参数的平方就可以忽略了，于是模型不会进一步将这个参数调整为0。
L1正则化计算公式不可导 L2正则化公式可导

TensorFlow中提供tf.contrib.layers.l1_regularizer和tf.contrib.layers.l2_regularizer分别实现L1和L2正则化。

滑动平均模型

滑动平均模型可以使模型在测试数据上更健壮. TF中提供 tf.train.ExponentialMovingAverage 来实现滑动平均模型.

在初始化 ExponentialAverage 时,需要提供一个衰减率(decay), 将用于控制模型更新的速度. ExponentialMovingAverage 对每一个变量会维护一个影子变量(shadow variable), 其初始值为相应变量的初始值, 每次运行变量更新时, 影子变量的值会更新为:

$$shadow_variable = decayshadow_variable + (1-decay)variable $$

decay决定了模型更新的速度, decay越大模型越趋于稳定, 一般会把decay设置成非常接近1的数(0.9999)

为了使得模型在训练前期可以更新得更快, ExponentialMovingAverage 还提供了 num_updates 参数来动态设置 decay 大小, 如果在 ExponentialMovingAverage 初始化时提供了 num_updates 参数, 那么每次使用的衰减率将是:

$$$ min { decay, \frac{1+num_updates}{10+num_updates} } $$$
