Synthesizing tasks for XLogo-Mini

In this study, we aim to evaluate the quality of automatically synthesized tasks and their solution codes for the category of XLogo Mini. We have picked 30 reference tasks and synthesized 6 tasks for each of them. We would like to survey the quality of each of these synthesized tasks compared to their reference task.

The survey has 3 steps and they are described below.

Step 1: Check Reference Task/Code

We present one of the reference tasks and its solution code on the left panel of the page. The task is presented below the “ Reference Task” header and its solution code is presented under the “Reference Code” header. Carefully view the task and its solution code.

Step 2: Check Synthesized Task/Code

We present a synthesized task and its solution code, for the given reference task, on the right panel. The task is presented below the “Synthesized Task” header and its solution code is presented under the “Synthesized Code” header. Carefully view the task and its solution code.

Step 3: Finish the Survey Questions

You are provided with a google sheet with the survey questions to be answered for each synthesized task. Each row in the sheet corresponds to a specific synthesized task. For each task, we present 6 questions. For each question, the rating can be picked from the given dropdown menu in each cell of the google sheet.

Here are the questions:

Q1: Overall rating of the synthesized task

  • -1 (Poor quality, task cannot be solved or is very confusing, a teacher won't present this task to students)
  • 0 (Satisfactory quality, a teacher would present this task to students with some modifications)
  • 1 (Excellent quality, a teacher can present this task to students without any adjustments)

Q2: Difficulty of the synthesized task w.r.t reference task

  • -1 (Easier than the reference task)
  • 0 (Similar difficulty to the reference task)
  • 1 (More difficult than the reference task)

Q3: Rating of the synthesized task in terms of concepts covered (e.g., shapes, colors, code structure, etc.) w.r.t. the reference task

  • -1 (Misses many important or relevant concepts that are essential)
  • 0 (Contains a few unnecessary or irrelevant concepts that are not included in the reference task.)
  • 1 (Includes the same concepts as the reference task, or contains additional necessary concepts)

Q4: If there are distractors in the synthesized task (e.g., walls, forbidden objects/grids), are they useful?

  • There are no distractors
  • -1 (Not useful, all the distractors are not helpful for achieving the learning objectives)
  • 0 (Partially useful, some distractors are useful while others may not be useful.)
  • 1 (Very useful, all distractors are essential for achieving the learning objectives)

Q5: Rating of synthesized code w.r.t reference code

  • -1 (Poor quality, synthesized code cannot solve the task, or violates the constraints in the task description)
  • 0 (Acceptable quality, the synthesized code can solve the task and satisfies the code constraints (if any), but contains unnecessary commands)
  • 1 (Good quality, the synthesized code can solve the task, satisfies the code constraints (if any) and doesn't include any unnecessary commands.)

Guidelines

1. Please make sure that the task id (e.g., task18.p5) on the webpage matches the task id on the google sheet, while filling in the survey for a specific task.

2. It would be more convenient to keep two pages open side-by-side: the webpage which describes each task and the google sheet to enter the rating for the task.

3. For each synthesized task, it typically takes 3-5 minutes to complete these three steps.

4. Kindly provide your "Overall Comments" in the google sheet (last column). Your comments will be valuable for us.
